# ✅ Чек-ліст тестування фінансових розрахунків

## 🎯 Перед покупкою активу:

1. **Запишіть початкові значення:**
   - [ ] Готівка (Cash): $**\_\_\_**
   - [ ] Пасивний дохід (Passive Income): $**\_\_\_**
   - [ ] Витрати (Expenses): $**\_\_\_**
   - [ ] Кількість активів: **\_\_\_**

## 💰 Після покупки активу:

2. **Перевірте зміни:**
   - [ ] Готівка зменшилася на суму downPayment
   - [ ] Пасивний дохід збільшився на cashFlow активу
   - [ ] Актив з'явився в списку активів
   - [ ] У консолі з'явилося: "💰 Recalculated finances for [ім'я гравця]"

## 🔄 Після завершення ходу:

3. **Перевірте місячні фінанси:**
   - [ ] З'явилося повідомлення з результатами місяця
   - [ ] Net Cash Flow = Salary + Passive Income - Expenses
   - [ ] Готівка оновилася на суму Net Cash Flow
   - [ ] Якщо Passive Income > Expenses - можна на швидку доріжку

## 📊 У фінансовій панелі:

4. **Перевірте відображення:**
   - [ ] Passive Income відображається правильно
   - [ ] Total Expenses включають професійні + зобов'язання
   - [ ] Прогрес-бар швидкої доріжки оновився
   - [ ] Cash Flow = Passive Income - Expenses

## 🚨 Критичні помилки:

- [ ] **НЕ має бути**: Passive Income рахується двічі
- [ ] **НЕ має бути**: Активи зникають після перерахунку
- [ ] **НЕ має бути**: Негативна готівка без автоматичного боргу

## 🎮 Тестові сценарії:

### Сценарій 1: Покупка недорогого активу

- [ ] Купити актив за ~$1000 з cashFlow ~$50
- [ ] Перевірити всі пункти вище

### Сценарій 2: Покупка дорогого активу

- [ ] Купити актив за ~$10000 з cashFlow ~$500
- [ ] Перевірити що швидка доріжка стала доступною

### Сценарій 3: Недостатньо грошей

- [ ] Спробувати купити актив без достатньої готівки
- [ ] Перевірити що з'явилася помилка
