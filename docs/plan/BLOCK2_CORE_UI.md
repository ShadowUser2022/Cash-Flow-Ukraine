# Блок 2 — Core UI

Ціль: мінімальний UI, який чітко показує, що відбувається у core-циклі.

## Must Have

- Lobby з іменем гравця. ✅
- Вибір мрії. ✅
- Видима активна дошка.✅
- Видима позиція гравця.✅
- Основна дія завжди одна: кинути кубик або вирішити pending подію.
- Deal має дві кнопки: прийняти / пропустити.
- Expense має дві кнопки: оплатити / в кредит.
- Візуально різні типи клітинок.
- Finance strip: cash, passive, expenses, dream.
- Fast Track progress.

## Поточне Рішення

- Не показуємо всі 24/16 клітинок одразу.
- Показуємо board window з 5 клітинок і підказками, що дошка продовжується.
- `було -> стало` прибрано з основного UI, щоб не перевантажувати екран.
- Хід пояснюється через guide panel, dice state і активну позицію.

## Перевірено

- Start game виглядає зрозуміло на desktop.
- Roll animation не плутає користувача.
- Pending deal не дублює інформацію.
- Pending expense не дублює інформацію.
- Board window дає відчуття довшої дошки.

## Ще Перевірити

- Start game виглядає зрозуміло на mobile.
- Fast Track transition має зрозуміле повідомлення.
- Win state має зрозумілу кінцеву дію.

## Остання Перевірка

- **Дата:** 2026-04-25
- **Метод:** browser pass на локальному frontend + API прогін backend.
- **Результат:** desktop core UI пройдено; mobile responsive CSS підправлено, але треба ще підтвердити в реальному mobile/Safari viewport.
