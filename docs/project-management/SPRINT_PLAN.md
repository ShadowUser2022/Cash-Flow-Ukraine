# 🎮 Cash Flow Ukraine - Sprint Plan

## 📅 ПОТОЧНИЙ СПРИНТ: "Player Connection & Game Lobby"
*Дати: 7-14 червня 2025 (1 тиждень)*

### 🎯 Sprint Goal
Реалізувати повний цикл підключення гравців до гри: від входу за Game ID до готовності почати гру в лобі.

---

## 📋 SPRINT BACKLOG

### 🟥 КРИТИЧНІ ЗАДАЧІ (MUST HAVE)

#### 1. 🎭 Player Join System
**Priority: P0 | Estimate: 2 дні**

- [ ] **Frontend: Форма входу в гру**
  - [ ] Інпут для Game ID (6-символьний код)
  - [ ] Інпут для імені гравця (валідація 2-20 символів)
  - [ ] Кнопка "Приєднатися до гри"
  - [ ] Обробка помилок (гра не знайдена, повна гра)

- [ ] **Backend: API для приєднання**
  - [ ] `POST /api/games/:gameId/join` endpoint
  - [ ] Валідація Game ID та імені гравця
  - [ ] Перевірка кількості гравців в грі
  - [ ] Додавання гравця до гри в in-memory storage

- [ ] **Socket.IO: Real-time приєднання**
  - [ ] Event `join-game` з gameId та playerName
  - [ ] Broadcast `player-joined` всім в грі
  - [ ] Обробка помилок підключення

**Acceptance Criteria:**
- ✅ Гравець може ввести Game ID та ім'я
- ✅ Система валідує дані та показує помилки
- ✅ При успішному приєднанні гравець потрапляє в лобі
- ✅ Інші гравці бачать нового гравця в реальному часі

#### 2. 🏢 Profession Selection 
**Priority: P0 | Estimate: 1 день**

- [ ] **Система професій**
  - [ ] Список доступних професій в константах
  - [ ] UI для вибору професії (dropdown або modal)
  - [ ] Автоматичне призначення при входженні
  - [ ] Можливість зміни до початку гри

- [ ] **Професії для MVP:**
  - [ ] Nurse (Медсестра) - $3,100/month
  - [ ] Teacher (Вчитель) - $3,300/month  
  - [ ] Truck Driver (Водій вантажівки) - $2,500/month
  - [ ] Police Officer (Поліцейський) - $3,000/month

**Acceptance Criteria:**
- ✅ Гравець може вибрати професію з списку
- ✅ Показуються базові параметри професії
- ✅ Всі гравці бачать професії один одного

#### 3. 🎪 Game Lobby Interface
**Priority: P0 | Estimate: 2 дні**

- [ ] **UI Лобі гри**
  - [ ] Відображення Game ID великими літерами
  - [ ] Список підключених гравців з професіями
  - [ ] Індикатори готовності кожного гравця
  - [ ] Налаштування гри (тільки для хоста)

- [ ] **Готовність гравців**
  - [ ] Чекбокс "Готовий до гри" для кожного гравця
  - [ ] Socket event `player-ready-toggle`
  - [ ] Блокування зміни професії після готовності
  - [ ] Візуальна індикація готових/не готових

- [ ] **Контролі хоста**
  - [ ] Кнопка "Почати гру" (активна тільки якщо всі готові)
  - [ ] Можливість видалити гравця
  - [ ] Зміна максимальної кількості гравців (2-6)

**Acceptance Criteria:**
- ✅ Лобі показує всю необхідну інформацію про гру
- ✅ Гравці можуть позначити готовність
- ✅ Хост може почати гру тільки коли всі готові
- ✅ Всі зміни відображаються в реальному часі

### 🟨 ВАЖЛИВІ ЗАДАЧІ (SHOULD HAVE)

#### 4. 🎲 Basic Game Start Transition
**Priority: P1 | Estimate: 1 день**

- [ ] **Перехід від лобі до гри**
  - [ ] Socket event `start-game` від хоста
  - [ ] Зміна статусу гри з 'waiting' на 'in-progress'
  - [ ] Перенаправлення всіх гравців на ігровий екран
  - [ ] Ініціалізація початкового стану гри

**Acceptance Criteria:**
- ✅ Після натискання "Почати гру" всі гравці переходять в гру
- ✅ Статус гри оновлюється в базі даних
- ✅ Гравці більше не можуть приєднуватися до розпочатої гри

#### 5. 💾 Game State Persistence
**Priority: P1 | Estimate: 1 день**

- [ ] **Збереження стану в in-memory storage (поки без MongoDB)**
  - [ ] Автоматичне збереження при змінах в пам'яті
  - [ ] Відновлення стану при перепідключенні (в межах сесії)
  - [ ] Cleanup завершених/покинутих ігор з пам'яті

**Acceptance Criteria:**
- ✅ Стан гри зберігається при кожній зміні
- ✅ Гравці можуть перепідключитися без втрати даних

## 📊 SPRINT METRICS

### Definition of Done:
- ✅ Код протестований та працює
- ✅ TypeScript без помилок
- ✅ API endpoints задокументовані
- ✅ UI responsive та user-friendly
- ✅ Socket.IO events працюють стабільно
- ✅ Error handling реалізований
- ✅ Code review пройдено

### Success Criteria:
- ✅ 2-6 гравців можуть приєднатися до гри
- ✅ Всі гравці бачать один одного в лобі
- ✅ Хост може почати гру коли всі готові
- ✅ Перехід від лобі до гри працює smooth
- ✅ Стан зберігається та відновлюється

### Risk & Mitigation:
- **Ризик:** Складність Socket.IO синхронізації
  - **Пом'якшення:** Почати з простих events, додати складні поступово

- **Ризик:** UI займе більше часу ніж планувалося
  - **Пом'якшення:** Зосередитися на функціональності, поліш потім
