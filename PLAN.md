## ПЛАН МІН — MVP (Березень 2026)

**Мета:** Повноцінна мультиплеєрна партія для 2-4 гравців без падінь та десинхронізації

### Тиждень 1 — Стабілізація базової петлі гри

- [ ] **Виправити продаж активів** — `GameMechanicsService.ts`: списати заставу, повернути cash, прибрати актив з passive income
- [ ] **Кнопка "Пропустити"** для SMALL DEAL та BIG DEAL — socket event `reject-deal`, після чого передати хід
- [ ] **BIG DEAL спрощення** — тільки поточний гравець вирішує купити/ні (аукціон між гравцями — в ПЛАН МАКС)
- [ ] **Автоматичне списання витрат на PAYDAY** — стабілізувати логіку в `GameMechanicsService.ts`
- [ ] **Auto turn timeout** — через 60 сек без дії автоматично передати хід

### Тиждень 2 — Надійність мультиплеєру

- [ ] **Відновлення з'єднання** — при реконнекті надіслати повний `game-state` без втрати прогресу
- [ ] **Обробка виходу гравця** — гра продовжується без краша, хід переходить далі
- [ ] **Десинхронізація** — виявлення та примусова синхронізація стану
- [ ] **Фінансовий звіт опонентів** — read-only перегляд балансу інших гравців

### Тиждень 3-4 — Шліфовка та деплой MVP

- [ ] **Тестування** — повна партія 2, 3, 4 гравці від початку до кінця
- [ ] **Екран завершення** — підсумок для всіх гравців (активи, cash flow, мрія)
- [ ] **Події в чаті** — "Іванко купив Квартиру за 50,000 грн", "Петро вийшов на Fast Track"
- [ ] **Деплой MVP v1.0** на Railway + Vercel

### Що ВИРІЗАЄМО для MVP (залишаємо на ПЛАН МАКС)
- Аукціон BIG DEAL між всіма гравцями (занадто складно + ризик десинхронізації)
- Торгівля активами між гравцями
- WebRTC відеочат
- MongoDB / база даних (залишаємо in-memory)
- Ринкові множники при продажу
- Мобільна версія

## Правила гри — послідовність

> Повні правила: `docs/documentation/mechanics/GAME_RULES_AND_MECHANICS.md`
> Тут — скорочена версія для орієнтиру в розробці

### Підготовка до гри
1. Кожен гравець обирає **професію** (визначає зарплату, витрати, стартову готівку)
2. Кожен гравець обирає **мрію** (ціль для перемоги на Fast Track)
3. Всі стають на клітинку START (позиція 0)

### Послідовність ходу на Rat Race

```
КРОК 1 — Кидок кубика (1-6) + рух за годинниковою стрілкою

КРОК 2 — Якщо ПРОЙШОВ через PAYDAY (позиція 0):
          → отримати зарплату
          → списати щомісячні витрати

КРОК 3 — Виконати дію клітинки:

  PAYDAY    → +зарплата, -витрати (якщо зупинився)
  SMALL DEAL → картка малої угоди:
               [Купити] → -down payment, +актив, +passive income
               [Пропустити] → нічого
  BIG DEAL  → картка великої угоди (MVP: тільки поточний гравець):
               [Купити] / [Пропустити]
  DOODAD    → обов'язкова витрата, -готівка
  BABY      → +500 грн до щомісячних витрат (назавжди)
  DOWNSIZE  → пропустити 2 наступних ходи
  CHARITY   → добровільно пожертвувати 10% зарплати
               → якщо так: кидати 2 кубики наступні 3 ходи
  MARKET    → подія на ринку (зміна вартості активів)

КРОК 4 — Перевірка фінансової свободи:
          якщо Пасивний дохід ≥ Витрати → ПЕРЕХІД НА FAST TRACK

КРОК 5 — Передача ходу наступному гравцю
```

### Послідовність ходу на Fast Track

```
КРОК 1 — Кидок 2 кубиків (2-12) + рух

КРОК 2 — Виконати дію клітинки:

  CASHFLOW DAY → +пасивний дохід × множник (2-5)
  DREAMS       → якщо готівка ≥ вартість мрії → ПЕРЕМОГА
  BUSINESS     → велика угода
  LAWSUIT      → штраф або втрата активів
  DIVORCE      → -50% готівки та активів
  TAX AUDIT    → -20% від загального доходу

КРОК 3 — Перевірка перемоги:
          готівка ≥ вартість мрії → ГРА ЗАКІНЧЕНА
```

### Фінансові формули

```
Cash Flow      = Зарплата + Пасивний дохід - Витрати
Фін. свобода   = Пасивний дохід ≥ Витрати  →  перехід на Fast Track
Перемога       = Готівка ≥ Вартість мрії   (тільки на Fast Track)
Банкрутство    = Готівка < 0 і немає активів для продажу
```

### Правила купівлі/продажу активів

- Купити можна тільки якщо готівка ≥ down payment
- Продати можна в будь-який момент свого ходу
- Після продажу: +готівка (ціна - залишок кредиту), -passive income
